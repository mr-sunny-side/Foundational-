# ex06 バグまとめ：コンポーネント分割

## バグ1: `TaskList.jsx` — `.map()` の書き方が間違い

**場所:** `TaskList.jsx` 7〜9行目

```jsx
// ❌ 間違い
Task.map(
    <li key={Task.id}>
        <span style={{textDecoration: Task.complete ? "line-through" : "none"}}>
            {Task.text}
        </span>
```

**問題点:**
- `Task`（単数形）は存在しない変数。propsで受け取っているのは `Tasks`（複数形）
- `.map()` にコールバック関数 `(task) => ...` を渡していない
- map内の変数も `Task` ではなくコールバック引数 `task` を使うべき

```jsx
// ✅ 正しい書き方
Tasks.map((task) => (
    <li key={task.id}>
        <span style={{textDecoration: task.complete ? "line-through" : "none"}}>
            {task.text}
        </span>
        <CompButton Tasks={Tasks} setTask={setTask} id={task.id}/>
        <DeleteButton Tasks={Tasks} setTask={setTask} id={task.id}/>
    </li>
))
```

## バグ2: `TaskItem.jsx` — `DeleteButton` / `CompButton` がJSXを返していない

**場所:** `TaskItem.jsx` 6〜19行目

```jsx
// ❌ 間違い
function DeleteButton({Tasks, setTask, id}) {
    Tasks.filter(task => task.id !== id);  // 結果をsetTaskに渡していない
};  // JSX（ボタン）を return していない

function CompButton({Tasks, setTask, id}) {
    Tasks.map(task =>
        task.id === id ? {...task, complete: !task.complete} : task
    );  // 結果をsetTaskに渡していない
};  // JSX（ボタン）を return していない
```

**問題点:**
- Reactコンポーネントは必ず JSX を `return` しなければならない（ボタンがない）
- `filter()` / `map()` の結果を `setTask()` に渡していないので状態が更新されない
- `function` 宣言の末尾に `;` はつけない

```jsx
// ✅ 正しい書き方
function DeleteButton({Tasks, setTask, id}) {
    const handleDelete = () => {
        setTask(Tasks.filter(task => task.id !== id));
    };
    return <button onClick={handleDelete}>削除</button>;
}

function CompButton({Tasks, setTask, id}) {
    const handleComplete = () => {
        setTask(Tasks.map(task =>
            task.id === id ? {...task, complete: !task.complete} : task
        ));
    };
    return <button onClick={handleComplete}>完了</button>;
}
```

## バグ3: `TaskItem.jsx` — `AddButton` がコンポーネントではなく、配列の `[]` が抜けている

**場所:** `TaskItem.jsx` 22〜27行目

```jsx
// ❌ 間違い
function AddButton({Tasks, setTask, Input, setInput}) {
    if (Input.trim() === "") return;
    const newTask = {id: Date.now(), text: Input, complete: false};
    setTask(...Tasks, newTask);  // [] が抜けている → 配列ではなく引数として展開してしまう
    setInput("");
    // JSX（ボタン）を return していない
}
```

**問題点:**
- JSX を `return` していないのでボタンが表示されない
- `setTask(...Tasks, newTask)` は `setTask` に複数の引数を渡している。`[...Tasks, newTask]` と `[]` で配列にする必要がある

```jsx
// ✅ 正しい書き方
function AddButton({Tasks, setTask, Input, setInput}) {
    const handleAdd = () => {
        if (Input.trim() === "") return;
        const newTask = {id: Date.now(), text: Input, complete: false};
        setTask([...Tasks, newTask]);
        setInput("");
    };
    return <button onClick={handleAdd}>追加</button>;
}
```

## まとめ

| ファイル | 問題 | 原因 |
|---|---|---|
| `TaskList.jsx` | `.map()` が動かない | 変数名のミス・コールバック関数の書き忘れ |
| `TaskItem.jsx` | ボタンが表示されない | JSXを `return` していない |
| `TaskItem.jsx` | 状態が更新されない | `filter()`/`map()` の結果を `setTask()` に渡していない |
| `TaskItem.jsx` | 追加が動かない | `setTask([...Tasks, newTask])` の `[]` が抜けている |

## 学んだこと

- `.map()` は `配列.map((要素) => JSX)` の形で書く
- Reactコンポーネントは必ず JSX を `return` する
- `filter()` / `map()` は元の配列を変更しないので、結果を `setTask()` に渡す
- スプレッド構文で配列をコピーするときは `[...配列]` と `[]` を忘れない
