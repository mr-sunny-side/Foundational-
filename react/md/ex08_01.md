# ex08 バグ修正メモ

`filtered_tasks.map is not a function` エラーの原因と修正箇所。

---

## バグ一覧

### バグ1（エラーの直接原因）: TaskList.jsx — デストラクチャリング漏れ＋prop名ミスマッチ

```jsx
// 間違い
function TaskList(filtered_tasks, set_task) {

// 正しい
function TaskList({filtered_task, set_task}) {
```

React コンポーネントの引数は props オブジェクト1つ。`{}` でデストラクチャリングしないと `filtered_tasks` に props オブジェクト全体が入り `.map()` できない。さらに ex08.jsx が渡す prop 名は `filtered_task`（単数）なのに `filtered_tasks`（複数）で受け取っていた。

---

### バグ2: TaskList.jsx — textDecoration のロジックが逆

```jsx
// 間違い（完了していないときに打ち消し線が入る）
style={{textDecoration: !task.complete ? "line-through" : "none"}}

// 正しい（完了しているときに打ち消し線が入る）
style={{textDecoration: task.complete ? "line-through" : "none"}}
```

---

### バグ3: ex08.jsx — `.map()` → `.filter()`

```jsx
// 間違い（boolean の配列になってしまう）
filteredTask = Tasks.map(task => !task.complete)
filteredTask = Tasks.map(task => task.complete)

// 正しい（タスクオブジェクトの配列を返す）
filteredTask = Tasks.filter(task => !task.complete)
filteredTask = Tasks.filter(task => task.complete)
```

`.map()` は各要素を変換した新しい配列を返す。フィルタリングには `.filter()` を使う。

---

### バグ4: ex08.jsx — TaskList に渡す prop が間違い

```jsx
// 間違い（set_filter を渡している）
<TaskList filtered_task={filteredTask} set_filter={setFilter}/>

// 正しい（削除・完了ボタンに必要な setTask を渡す）
<TaskList filtered_task={filteredTask} set_task={setTask}/>
```

---

### バグ5: TaskItem.jsx — スプレッド構文の配列リテラル漏れ

```jsx
// 間違い（配列ではなく引数として展開してしまう）
set_task(...tasks, new_task);

// 正しい（配列として渡す）
set_task([...tasks, new_task]);
```

`useState` の setter には配列を1つ渡す必要がある。

---

### バグ6: TaskItem.jsx — DeleteButton で `.map()` → `.filter()`

```jsx
// 間違い（boolean の配列になってしまう）
set_task(tasks.map(task => task.id !== id));

// 正しい
set_task(tasks.filter(task => task.id !== id));
```

---

## まとめ

| ファイル | バグ | 修正 |
|---|---|---|
| TaskList.jsx | デストラクチャリング漏れ・prop名ミスマッチ | `{filtered_task, set_task}` に変更 |
| TaskList.jsx | textDecoration のロジックが逆 | `!task.complete` → `task.complete` |
| ex08.jsx | フィルタリングに `.map()` を使用 | `.filter()` に変更 |
| ex08.jsx | TaskList に `set_filter` を渡している | `set_task={setTask}` に変更 |
| TaskItem.jsx | スプレッド構文で配列リテラルが漏れ | `[...tasks, new_task]` に変更 |
| TaskItem.jsx | 削除処理に `.map()` を使用 | `.filter()` に変更 |
